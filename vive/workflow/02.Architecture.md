\# Architecture Specification (Final)

\#\# 0\. 기본 전제

\- 본 프로젝트는 \*\*항상 2개 이상의 산출물(output)\*\* 을 가진다.  
\- \*\*랜딩 페이지는 SEO 목적상 반드시 포함\*\*된다.  
\- 모바일 앱, 데스크탑 앱, 웹 앱은 모두 \*\*클라이언트\*\*다.  
\- 서버의 주된 비즈니스 로직은 \*\*단일 API 백엔드\*\*에서 처리하며,  
  모든 클라이언트는 이를 \*\*공유\*\*한다.

\---

\#\# 1\. 최상위 아키텍처 원칙

\- 최상위 디렉토리는 역할(frontend / backend)이 아니라    
  \*\*실행 · 배포 · 릴리즈 단위\*\*를 기준으로 구성한다.  
\- 실행 단위는 \`apps/\` 디렉토리에 위치한다.  
\- 재사용 코드는 실행 단위가 아니며, 별도의 영역으로 분리한다.  
\- 구조는 단기 구현 편의보다 \*\*장기 운영 안정성\*\*을 우선한다.

\---

\#\# 2\. 최종 디렉토리 구조

\`\`\`txt  
.

├── packages/                 \# 공통 재사용 코드 (비실행)  
│   ├── ui/                   \# 공통 UI 컴포넌트

│
