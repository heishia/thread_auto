

\# 문서 생성 규칙

하단의 문서 구조를 생성, 그 이후 초기에 생성한 문서 수를 유지  
추가 문서를 절대로 생성하지 않는다 (토큰 사용금지) → 프로젝트 룰에 추가   
기존 문서를 업데이트 하는 방식으로 진행한다.

\#\#  문서 구조

project-root/  
├─ README.md \# 프로젝트 진입점 (무조건 있음)  
├─ docs/  
│ ├─ 00\_overview.md  
│ ├─ 01\_requirements.md  
│ ├─ 02\_domain.md  
│ ├─ 03\_usecases.md  
│ ├─ 04\_architecture.md  
│ ├─ 05\_api.md  
│ ├─ 06\_dev\_guide.md

\#\#  README.md는 “설명서”가 아니라 \*\*입구\*\*

\#\#\# README.md 역할  
\- \*\*처음 들어온 사람(= 미래의 나)\*\* 이    
\- \*\*“이게 뭐고, 어떻게 실행하지?”\*\* 를    
\- \*\*3분 안에 알게 하는 문서\*\*

\#\#\# 작성 원칙  
\- 깊게 설명하지 않는다    
\- 필요 내용은 \*\*docs 폴더 문서 링크 제공\*\*  
\- \*\*아키텍처 다이어그램 한 장 넣으면 최고\*\*
