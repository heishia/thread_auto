\# SEO 적용 워크플로우 (Naver \+ Google)

본 문서는 서비스 운영의 검색 노출 최적화를 목적으로 하며, 검색 엔진(Naver, Google)이 웹사이트를 안정적으로 수집하고 노출할 수 있도록 설계된 실행 기준을 정의한다.

\---

\#\# 1\. 기본 전제

\- Next.js 기반 SSR 또는 SSG 방식을 우선 적용한다.  
\- 주요 페이지는 HTML 형태의 콘텐츠를 서버에서 직접 전달해야 한다.  
\- 검색봇이 단일 자원(JS, CSS, 이미지)을 수집하는 것을 허용한다.  
\- JavaScript 기반 리다이렉트는 사용하지 않는다.

\---

\#\# 2\. URL 정책

\#\#\# 2.1 단일 URL 강제  
\- www와 non-www 중 하나로 통일하고 HTTP 301 리다이렉트만 사용한다.  
\- URL 체계 변경 시 302는 임시 상황에서만 사용한다.  
\- 301 체인은 두 단계 이상 늘리지 않는다.

\#\#\# 2.2 의미 기반 URL 유지  
\- 파라미터 기반 경로 사용 제한  
\- Hash 기반 라우팅 금지  
\- 구글 및 네이버가 페이지 목적을 URL로 이해할 수 있도록 명료한 표현 사용

\---

\#\# 3\. 검색 엔진 접근 허용

\#\#\# 3.1 robots.txt  
\- 모든 공개 페이지에 대한 크롤링 허용  
\- 관리자, 인증, 내부 API, 비공개 문서는 차단  
\- JS/CSS/Image 수집 차단 금지  
\- sitemap 위치 명시

\#\#\# 3.2 sitemap.xml  
\- 모든 공개 페이지 포함  
\- 페이지 추가/삭제 시 자동 반영  
\- 수동 관리 또는 정적 sitemap 사용 금지

\---

\#\# 4\. HTML 메타 정책

\#\#\# 4.1 Canonical  
\- 모든 URL에 canonical 지정  
\- Querystring 포함 canonical 금지

\#\#\# 4.2 SNS Preview 메타  
\- Open Graph 필수  
\- Twitter Card 적용 권장

\#\#\# 4.3 Structured Data  
\- WebSite, Organization 구조는 기본 포함  
\- 향후 Article, Breadcrumb 포함 확장 고려

\---

\#\# 5\. 렌더링 정책

\- SSR 기반 렌더링 원칙 적용  
\- 주요 컨텐츠는 서버에서 문자열 형태로 표현  
\- Lazy-loading은 fold 아래에서만 사용  
\- 무한 스크롤 방식은 반드시 링크 기반 목록 페이지를 병행

\---

\#\# 6\. 리다이렉트 정책

\#\#\# 허용 방식  
\- HTTP 301: 영구 이동  
\- HTTP 302: 임시 이동  
\- JavaScript 기반 이동 금지

\#\#\# 제한 사항  
\- 차단된 페이지로 이동 금지  
\- robots.txt 금지 대상 목적지로 이동 금지  
\- 순환 리다이렉트 발생 시 즉시 수정

\---

\#\# 7\. 구글 SEO 보강

\#\#\# 7.1 인덱싱 관리  
\- Google Search Console 등록  
\- URL 제출 및 상태 모니터링  
\- Coverage 오류 즉시 수정

\#\#\# 7.2 Core Web Vitals 준수  
\- LCP, CLS, INP 기준 점검  
\- 이미지는 적절한 preload 사용  
\- heavy script 최소화

\---

\#\# 8\. 랜딩 페이지 정책

\#\#\# 8.1 랜딩페이지 역할  
\- 사이트의 대표 컨텐츠를 명확히 안내한다.  
\- 검색엔진이 의미 있는 페이지로 인식하도록 고정 콘텐츠를 유지한다.  
\- 주요 키워드 기반 소개 문구를 포함한다.

\#\#\# 8.2 필수 구현 규칙  
\- apps/landing 내부에 정적 노출 가능한 content 페이지를 유지한다.  
\- content 페이지는 수집 가능한 정보가 포함된 형태여야 한다.  
\- content 페이지는 검색 최적화 대상 우선순위 페이지로 간주한다.

\---

\#\# 9\. 수집 콘텐츠 정책

\#\#\# 9.1 content 페이지 필요성  
\- 검색 엔진이 텍스트 기반 콘텐츠를 수집할 수 있도록 항상 존재해야 한다.  
\- 랜딩 페이지에서 콘텐츠가 부족한 경우 해당 페이지가 검색 대상이 되지 않을 가능성이 있다.

\#\#\# 9.2 콘텐츠 생산 요구  
\- 서비스 운영자는 주기적으로 신규 텍스트 컨텐츠를 추가한다.  
\- 콘텐츠 제작 시 사용자로부터 키워드를 요구하는 과정을 필수 포함한다.  
\- 작성 규칙은 콘텐츠 제작 가이드 문서를 따른다.

\---

\#\# 10\. 운영 점검

\#\#\# 배포 시 확인사항  
\- sitemap 최신화  
\- canonical 누락 여부 확인  
\- noindex 불필요 활성화 발생 여부 점검

\#\#\# 월간 점검  
\- Search Console 및 네이버 웹마스터 도구 상태 확인  
\- 인덱스 오류 또는 제외 페이지 검토  
\- 컨텐츠 추가 여부 확인

\---

\#\# 11\. 목적

\- 네이버 및 구글 검색 서비스에서 공통된 기준으로 수집 및 노출  
\- URL·콘텐츠·렌더링을 통한 검색 품질 유지  
\- 신규 페이지 추가 시 일관된 SEO 적용
